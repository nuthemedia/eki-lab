# Design

## UI Direction

- 現代的で静かな学習体験。勉強アプリにも占いサイトにも寄せない
- 明るいニュートラル背景 + 墨色の線 + 1アクセント色。黒・茶・金の「占い色」を避ける
- 語彙は「つくる」「そろえる」「ひらく」「つぎへ」「もう一度つくる」。テスト・スコア・✗○は使わない
- タップターゲット44px以上、陰陽ボタンは片手親指で届く下部配置

## Screens

単一ルート `/hakke`(クライアント側ステートマシン、フェーズ: intro / build / nature / reveal / complete):

1. イントロ — タイトル + 一言 + 「つくる」
2. つくる画面 — 上部にお手本(淡い)、中央ステージ(canvas 背面 + 組み立て中の三本線 前面)、下部に「陽」「陰」ボタン、上端に進行ドット8個
3. 結果カード — 完成形 + 漢字 + 読み + 属性 + 自然イメージ一文の段階リビール + 「つぎへ」
4. 完成画面 — 8卦グリッド + 「もう一度つくる」

## Responsive Behavior

- モバイルファースト: `max-width: 480px` の中央カラム、`100dvh`、`clamp()` によるフルード指定、safe-area inset 対応(iching と同方式)
- デスクトップでは同じカラムが中央に浮くだけ。専用レイアウトは持たない

## Motion And Effects

- 爻の配置: 下からスライドイン(motion/react)。お手本と違う爻は一瞬現れて静かに溶けて消える(エラー表現なし)
- 結果カード: 漢字 → 属性 → イメージ文の順に段階フェードイン
- 自然現象ステージ: Canvas 2D、卦ごとに約2秒の決定的(シード乱数)演出。パーティクル上限200、dpr上限2、非再生時は rAF 停止。震のシェイクは canvas 内 translate のみで DOM は揺らさない
- `prefers-reduced-motion`: スライド/ステージ演出を省略し、現象は最終フレームの静止表示のみ
