# 易のかたち Project

ユーザーの曖昧な悩みを、易が答えられる「問い」へ整え、その問いに対して卦を立て、現代の言葉で読み解くアプリ。

Route: `/ask`(メインフロー)、`/formal`・`/dice`(問いなしのクイック立卦)、`/hexagrams`・`/hexagrams/[number]`(64卦AI辞典)、`/card`・`/card/r/[code]`(卦カード=ライト入口+SNS共有)

`/`はAWAI Commonsの紹介ページであり、この母体機能の入口ではない。母体ルートは削除せず直接URLで利用できる状態を維持するが、トップ導線・sitemap・検索結果からは外す(`noindex, follow`)。公開プロダクトはTaikyoku・Hakke・コイン易。

## Purpose

体験の核は **悩みを書く → 問いが整う → 卦を立てる → 今どう向き合うかが見える**。

- すぐに答えを出さない。実際の易者の「筮前の審事」のように、まず LLM で問いを整える
- 卦は、ユーザーが「この問いで占う」と明示してから立てる
- 未来を断定しない。「当たる/外れる」より「今どう向き合うか」を重視する
- 神秘性はあるが煽らない。占いアプリではなく、問いを深めるための易の道具

## Audience

- 悩みはあるが、何を知りたいのか自分でも整理できていない人
- 既存の占いサイトの安い演出・断定口調が苦手な人
- 道具(筮竹・算木・骰子)の美しさ・儀式感に惹かれる人

## MVP Scope

- 悩みの自由入力(500字まで)→ LLM による問い整理(要約・カテゴリ・占的候補3〜4・必要なら確認質問1つ)→ 候補選択 or 自分で編集 → 最終占的の確認 → 立卦モード選択 → 立卦演出 → LLM 解釈 → 保存
- 立卦は本物のロジック: 本格モード=本筮の確率(老陰1/16・少陽5/16・少陰7/16・老陽3/16)、サイコロモード=下卦1/8・上卦1/8・変爻1/6(変爻は常に1本)
- 解釈は固定データ(卦辞・爻辞・現代語訳)を LLM が問いに沿って橋渡しする。断定禁止・依存防止のトーンをシステムプロンプトと UI 固定文の両方で担保
- LLM は OpenAI(問い整理= gpt-5-nano、解釈= gpt-5-mini)。**キー未設定・失敗時はテンプレートにフォールバックし、フローは必ず完走する**
- 保存・履歴は localStorage(端末内、最大50件)。履歴から保存済み解釈を再読できる(LLM 再呼び出しなし)
- 問い整理はセッション3回まで、API は端末別に日次10回のクォータ
- クイック立卦ページ(/formal, /dice)は問いなし・LLM なしで従来どおり動く
- 64卦AI辞典(/hexagrams): 64卦を眺め・調べ・理解する静的辞典。一覧カード+詳細ページ(卦辞・爻辞・AI書き下ろし解説・関係ナビ=互卦/之卦/錯卦/綜卦)。解説は全部事前生成(書き下ろし)で、ランタイム LLM 呼び出しなし・新 API ルートなし
- 卦カード(/card):「いまの自分に、卦を一枚」。ライトユーザー向けの入口。プリセットの問い候補5つ(シャッフル可)から1つ選ぶ → 短い立卦演出(castDice) → 静的コピーの卦カード。9:16画像の保存/Web Share(Instagram・TikTok向け)と、動的OG付き共有URL(X向け)で拡散し、本編 `/ask` へ送客する。**ランタイム LLM 呼び出しなし**

## Out Of Scope

- ユーザーアカウント、サーバー保存
- メインフロー(/ask)での共有機能・シェア画像生成(共有は卦カード /card のみが担う。カードの共有URLはコード=質問ID+卦番号+変爻位置から決定的に再現し、サーバー保存はしない)
- 辞典での「この卦に問いかける」対話機能(辞典は静的データのみ)
- 用九・用六、爻位関係(応・比など)を使った本格的な占断ロジック
- 会話型の継続カウンセリング(LLM 呼び出しは問い整理と解釈の2箇所のみ)
- 医療・法律・投資等の専門的助言(注意文で常に専門家への相談を促す)
