# 易のかたち Design

## 世界観

- キーワード: 静かな儀式 / 美しい道具 / 高級感 / 木・竹・和紙・石・墨 / 現代的ミニマル / 神秘的だが安っぽくない
- 目指す印象: 実際の道具が机上で静かに動く。易に向き合う儀式感。見ているだけで気持ちよく、触ってみたくなる。
- 避けるもの: 派手なスピリチュアル風、紫グラデーション、金ピカ、占いサイトっぽい安い演出、過剰な発光、和風記号の乱用。

## パレット

iching 専用の CSS 変数を `app/(core)/iching.css` に定義する(`app/globals.css` は変更しない)。

| 変数 | 値 | 用途 |
| --- | --- | --- |
| `--ik-bg` | `#17161a` | 背景の深い墨色 |
| `--ik-bg-soft` | `#211f24` | 面(カード・舞台)のわずかな持ち上げ |
| `--ik-paper` | `#e9e2d0` | 和紙色。主要テキスト |
| `--ik-paper-dim` | `#b9b2a1` | 補助テキスト |
| `--ik-muted` | `#7d786d` | 弱いテキスト・ラベル |
| `--ik-line` | `rgba(233, 226, 208, 0.14)` | 罫線・枠 |
| `--ik-bamboo` | `#c9a86d` | 筮竹のハイライト |
| `--ik-bamboo-deep` | `#8f7448` | 筮竹の影側 |
| `--ik-wood` | `#8a6a4a` | 算木の木肌 |
| `--ik-wood-deep` | `#5d452f` | 算木の影側 |
| `--ik-vermilion` | `#b0492f` | 変爻の朱(辰砂)。唯一の強調色 |

- 背景には和紙の質感を思わせる極薄いラジアルグラデーションを重ねる。ノイズ画像・発光は使わない。
- 影は柔らかく低コントラスト。`box-shadow` は黒の低 alpha のみ。

## AWAI Commonsトップ

トップ `/` は母体機能のサービス一覧ではなく、AWAI Commonsと公開プロダクトを紹介する明るいブランドページ。背景`#f5f5f0`、白い面、墨色`#2b2e38`、青紫`#5560d8`、ゴシック主体、16〜24px角丸、ピル型CTAを使う。サブページ(/ask, /formal, /dice, /hexagrams, /card)は既存ダークテーマのまま。

- 構成: 単一ブランドヘッダー → ヒーロー → 64卦ホイール → 視点パネル → プロダクトセクション「易を学ぶ 四つの入口」+学習フロー「学ぶ→つくる→占う→読む」+4カード → 「なぜ今、易を学ぶのか。」 → Note → フッター(X・コピーライト)
- 64卦ホイール(`components/core/home/HexagramWheel.tsx`, client): 外輪=64卦ミニ卦画+序卦番号(5.625°間隔)、内輪=八卦シンボル、中央=選択中の卦(卦画・卦名・読み・essence 一言)。トップでは`showDetailLink={false}`とし、ホイール内の詳細リンクは出さない。辞典へは独立した公開プロダクトカードから進む。ドラッグ、スナップ、卦タップ、矢印キー、`role="slider"`+`aria-valuetext`を維持する
- 視点パネル: 見出し「易から見えてくるもの」は白い角丸面の外へ置き、面の中で「偏りを見る」「現在地を知る」「変化の向きを読む」を3列のピクトグラムで示す
- プロダクトカード: 左に小さな識別ビジュアル、中央に正式名称と最大2行の説明、右に小型ピルCTAを置く横長行。「学ぶ→つくる→占う→読む」の順とカード順を一致させる。見出し・フロー・カード一覧は同じセクション内で8px間隔
- ホイールの3D表現: CSS 3D の傾いた円盤(perspective 900px、rotateX 26°、低alpha黒の楕円落ち影)。中央の卦情報は3D の外で直立し「円盤の上に浮かぶカード」(暖白ラジアルのベール、発光ではない)。hover 環境ではポインタに ±3〜4° の微パララックス。ドラッグ角度は射影後 rect で正規化した atan2 で逆射影
- ゆっくり自転: 非操作時に約1.2°/s で回り続け、中央の卦も静かに移ろう(「変化は止まらない」の象徴)。ドラッグ/スナップ中・focus・hover 中は停止、操作後4秒で再開。`prefers-reduced-motion` では自転しない。ホイール直下に microcopy「まわして、六十四の変化をたどる」
- 非公開母体機能(/ask, /formal, /dice, /card)のリンクはトップへ戻さない

## タイポグラフィ

- 卦名・見出し: 明朝系 `"Hiragino Mincho ProN", "Yu Mincho", "Noto Serif JP", serif`(`--ik-font-serif`)。字間広め(`letter-spacing: 0.12em` 前後)。
- UI・本文: システムゴシック(globals.css の `--font-body` と同系)。
- 英字サブラベルは小さく、トラッキング広く、`--ik-muted`。

## レイアウト

- スマホファースト。縦1カラム、コンテンツ最大幅 480px 中央寄せ、`min-height: 100dvh`。
- 上部: 舞台(道具のアニメーション領域、高さ固定)/ 中部: 卦(算木)の積み上がり / 下部: 進行ラベルと再生コントロール。
- フォントサイズ・余白は `clamp()` で流体化。

## モーション

- ライブラリは `motion`(`motion/react`)。速すぎず、静かに。大げさな演出・バウンスのしすぎは避ける。
- 算木が置かれる瞬間は低 stiffness の spring で「とん、と着地」する軽い達成感。
- 変爻は縁と markが朱色にほのかに変わる(発光させない)。
- 本卦 → 之卦は変爻の算木が rotateX でフリップして陰陽が入れ替わる。
- サイコロは CSS 3D の立方体が回転 → 減速 → 静止。
- `prefers-reduced-motion: reduce` ではアニメーションを短縮・省略する。

## フロー画面のトーン

- 最初から結果を急がせない。入力画面は広く余白を取り、テキストエリアは静かな枠線のみ
- 問いの候補はカード、最終占的は明朝で大きく中央表示
- 解釈文はセクションカードで短く読みやすく。断定・煽りの語彙は使わない(禁止/推奨表現は API ルートのシステムプロンプト参照)
- ローディングは明朝の一行が静かに明滅するだけ(スピナー・派手な演出なし)

## 画面構成

1. `/` AWAI Commonsトップ: ブランドヘッダー → ヒーロー → インタラクティブ64卦ホイール → 3視点の圧縮パネル → 見出し・学習フロー・横長4カードをまとめたプロダクトセクション → 「なぜ今、易を学ぶのか。」 → Note → フッター。ページはサーバーコンポーネント、ホイールのみクライアント島。
2. `/ask` メインフロー: 悩み入力 → 問い整理 → 立卦 → AI解釈(トーンは「フロー画面のトーン」参照)。
3. `/formal` 本格モード: 導入(「問いを心に定めてください」「五十策を前に置きます」「一本を太極として除きます」)→ 筮竹の舞台 → 算木積み上げ → 結果。
4. `/dice` サイコロモード: 導入(「問いを定め、出目に身を委ねます」「下卦・上卦・変爻を求めます」)→ サイコロの舞台 → 算木積み上げ → 結果。
5. 結果画面(両モード共通 `HexagramResultView`): 本卦と之卦を並べ、変爻を朱で示す。「もう一度」導線。

## 64卦AI辞典の画面

- 一覧は **CSS のみ**(motion/react 禁止)。卦画は軽量な `HexagramGlyph`(1爻=1div、陰爻は linear-gradient の中央空き)を使い、`HexagramFigure`(motion 依存)は占いフロー専用のまま
- カードの「立体感」は層状の box-shadow +紙の起伏を思わせる微グラデのみ。3D tilt・JS パララックス・発光はしない
- 朱(`--ik-vermilion`)の用途は従来どおり変爻関連のみ(六爻リストの動爻→之卦リンクなど)
- sticky ヘッダーは背景 `--ik-bg` ベタ+下ボーダー `--ik-line` で控えめに
- 辞典のスタイルは `app/(core)/iching.css` 末尾の辞典セクションに追記(新規 CSS ファイルは作らない)

## 卦カードの画面(/card)

- タグライン「いまの自分に、卦を一枚」。世界観・パレットは本編と同一(墨・和紙・竹・朱)。ポップにしない=ライト向けでも安い占い演出は禁止のまま
- カード(`HexagramCard` / `.ik-card`): `--ik-bg-soft` 面+`--ik-line` 枠+極薄いラジアル。上下にブランド行(易のかたち / eki-lab、タグライン / URL)を置き、スクショされても出所がわかる構成。中身は 問い(明朝・鉤括弧)→ 卦画(`HexagramFigure`、変爻は朱)→ 卦名・読み → keyword(竹色・明朝)→ message → 変爻の爻辞(朱ラベルの左罫カード)
- 問い候補ボタン(`.ik-card-qbtn`): カテゴリの小ラベル+問い本文。ホバーで竹色の枠
- 立卦演出は短く(2〜3秒)。六爻の積み上がり+変爻の朱のみ。スピナー禁止、明朝一行の明滅で待たせる
- 共有画像(9:16 / OG)も同じ配色・同じ構成要素。ImageResponse 側は `lib/ichingCardOg.tsx` の色定数を使い、CSS 変数とずらさない

## 道具の表現

- 筮竹: SVG の細長い棒(丸端・竹色の縦グラデ)。束は数十本の重なりで表現し、50本を厳密に描かない。
- 算木: 横棒の div。木色の縦グラデ + ごく薄い木目(線形グラデの繰り返し)+ 柔らかい影。陽爻は一本、陰爻は中央に空きのある二本。
- サイコロ: 易占い専用の構成。下卦・上卦用は `preserve-3d` の八面体(三角フェイス×8、八卦の漢字)、変爻用は立方体(初〜上の漢字)。いずれも骨・象牙を思わせる生成り色の面 + 墨色の字。
- 卦辞・爻辞: 結果画面に枠線カードで表示。原文は明朝・字間広め、現代語訳は小さめのゴシックで `--ik-paper-dim`。変爻の爻辞ラベルのみ朱。
