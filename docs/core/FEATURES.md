# 易のかたち Features

## トップ `/`

AWAI Commonsの紹介ページ。非同期サーバーコンポーネント+ホイールのみクライアント島。

構成: ブランドヘッダー → 大テーマ「易の知恵を、暮らしに生かす」+日英タグライン → **64卦ホイール** → 3視点の圧縮パネル → 「易を学ぶ 四つの入口」+学習フロー「学ぶ→つくる→占う→読む」+公開プロダクト4カード(易有太極 `/taikyoku` / HAKKE `/hakke` / コイン易占い `/coin` / 易経・六十四卦辞典 `/hexagrams`)→ 「いま易を学ぶ理由」+「はじめての方へ」 → RSS最新2件の「読みもの」 → フッター。

Note RSSはサーバー側で取得して1時間キャッシュする。`note.com/awaicommons/n/`の記事だけをRSS順で最大2件表示し、固定の「はじめての方へ」と重複URLを除外する。取得・解析失敗時は記事行だけを省略し、固定リンクとNote一覧リンクは常に残す。

ホイール(`components/core/home/HexagramWheel.tsx`): データはページがサーバー側で構築(`HEXAGRAMS_BY_NUMBER`+`linesOfHexagram`+`HEXAGRAM_DICTIONARY[n].essence`)。CSS 3D の傾いた円盤(rotateX 26°+hover パララックス)で、非操作時は約1.2°/sでゆっくり自転。ドラッグ、5.625°グリッドへのスナップ、卦タップ、矢印キー、`role="slider"`、`prefers-reduced-motion`に対応する。トップでは`showDetailLink={false}`としてホイール内の詳細リンクを表示せず、辞典へは公開プロダクトカードから進む。

視点パネルは静的なサーバー描画で、「易から見えてくるもの」の見出しを面の外へ置き、「偏りを見る」「現在地を知る」「変化の向きを読む」の3項目を白い角丸面の横一列に収める。独立したプロダクトセクションには見出し、学習フロー、左ビジュアル・中央本文・右CTAの横長カードを8px間隔で表示する。

非公開母体機能(`/ask`・`/formal`・`/dice`・`/card`)は直接URLで動作を維持するが、トップ導線・sitemap・検索対象から外す。

## メインフロー(`/ask`、`IchingFlow` のステートマシン)

```
input(悩み入力) → refining(問い整理API) → candidates(要約+占的候補
  ↳ needsClarification 時は確認質問1つ) → confirm(最終占的の確認)
  → mode(立卦モード選択) → casting(演出再生) → interpreting(解釈API)
  → result(解釈8セクション+保存)
```

- 悩み入力: 500字まで。「問いを整える」で refine API へ。下部に履歴と、問いなしクイック立卦(本格/サイコロ)への導線+トップ `/` への quiet link
- 候補: カード3〜4枚。「自分で問いを編集」「もう一度整理してもらう」(問い整理はセッション3回まで)
- 確認: 最終占的を大きく表示。「この問いで占う」で初めて立卦へ
- 立卦: モード選択と同時に `castYarrow()` / `castDice()` で結果を確定し、既存の演出プレイヤーを埋め込み再生(`onResult` でフローに戻る。「もう一度」ボタンは出さない=再筮しない)
- 解釈画面: 問い / 出た卦(本卦→之卦) / 卦の要点 / **問いへの答え(卦の結果を根拠に問いへ正面から答える最終回答文。強調カード)** / 本卦の読み / 変爻の読み / 之卦の読み / 今どう向き合うか / 注意点。「この結果を保存」「新しい問いを立てる」
- API が使えないときは全ステップがテンプレートで完走し、「簡易整理/簡易解釈で表示しています」と静かに注記する
- 履歴: localStorage の保存済み結果を一覧表示、タップで再読、×で削除

## 共通

- 各モードは `SequenceStep[]`(演出ステップ配列)を `useSequencePlayer` で再生する。
- 導入画面はユーザーのタップ(「はじめる」)で開始。タップ時に `pickReading()` でモック結果を1本選び、そこからステップ配列を生成する(SSR と乱数のハイドレーション不整合を避けるため、乱数はイベントハンドラ内でのみ使う)。
- 再生コントロール(`PlaybackControls`): 再生 / 一時停止・進める(次ステップへ)・スキップ(結果へジャンプ)・もう一度(新しい結果で最初から)・速度切替(通常 / 速い)。
- 進行ラベル: 六爻の生成中は「初爻・二爻・三爻・四爻・五爻・上爻」を表示。
- 算木の積み上げ: 一爻決まるたびに `HexagramFigure` の下から算木が追加される。老陽・老陰(変爻)は朱の縁とマークで識別できる。

立卦はモック廃止済みで、`domain/iching/cast.ts` の本物の乱数ロジックを使う(乱数はイベントハンドラ内でのみ実行)。クイックページ(/formal, /dice)は問いなし・LLM なしで従来の演出+卦辞・爻辞表示。

## 本格モード `/formal`

ステップフロー(`domain/formalModeSteps.ts` が `HexagramResult` から生成):

1. 導入: 「問いを心に定めてください」「五十策を前に置きます」「一本を太極として除きます」
2. `remove-one`: 筮竹の束から一本が静かに外へ移動。「太極として一策を除く」表示。
3. 六爻それぞれについて:
   - `split`: 束が左右二つに分かれる
   - `count`: 四本ずつ整理される簡略アニメーション
   - `line-built`: 爻(陽 / 陰 / 老陽 / 老陰)が決まり、算木が一段置かれる
4. `primary-complete`: 六爻が揃い、本卦名を表示
5. `changing-line`: 変爻をハイライト(変爻なしのシナリオではスキップ)
6. `transform`: 変爻が反転し之卦へ変化
7. `result`: 結果表示(`HexagramResultView`)

## サイコロモード `/dice`

易占い専用サイコロの構成: 八卦の漢字(乾兌離震巽坎艮坤)が入った**八面体×2**(サイコロ1=下卦、サイコロ2=上卦)+ 初〜上の漢字が入った**六面体×1**(サイコロ3=変爻)。出目と卦・変爻の対応は本物(結果自体はモック)。この方式は必ず変爻が1つ出るため、変爻なしシナリオは使わない。

ステップフロー(`domain/diceModeSteps.ts` が生成):

1. 導入: 「問いを定め、出目に身を委ねます」「下卦・上卦・変爻を求めます」
2. `roll-die`(1個目)→ `trigram`(下卦を明示表示)
3. `roll-die`(2個目)→ `trigram`(上卦を明示表示)
4. `roll-die`(3個目)→ `changing-line`(初爻〜上爻のどこが変化するか)
5. `primary-complete`: 上卦・下卦から本卦の六爻が算木として下から積み上がり、本卦名を表示。変爻を強調
6. `transform`: 変爻が反転し之卦へ変化(アニメーション)
7. `result`: 結果表示

※ 出目→卦・変爻の対応は本物(八面体の面=八卦 index、六面体の面=爻位)。結果の選択自体はモック。

## 64卦AI辞典 `/hexagrams`

64卦を眺め・調べ・理解する静的辞典。全データは事前生成(書き下ろし)で、LLM 呼び出し・API ルートはない。

### 一覧 `/hexagrams`

- 2列カードグリッド(480px フレーム内)。カード = 第◯卦 / 卦画(`HexagramGlyph` sm)/ 卦名(明朝)/ 読み / keywords 1行。タップで詳細へ
- フィルタ: 上卦・下卦の八卦チップ(☰乾 …)+ テキスト検索(卦名・読み・キーワードの部分一致)
- 表示切替: 8×8マトリクス(上=上卦・左=下卦の伝統配列。セルは卦画のみ、タップで詳細へ)
- sticky ヘッダー(タイトル+検索+切替)。0件時は静かな空表示
- 一覧は motion/react を使わない(CSS のみ。64カードの DOM を軽く保つ)

### 詳細 `/hexagrams/[number]`(1-64、静的生成)

上から:

1. ヒーローカード: 第◯卦 / 卦画(lg)/ 卦名 / 読み / 上卦×下卦(八卦の symbol+nature)/ keywords / essence。立体感は層状の影のみ(3D tilt なし)
2. 卦辞: `HEXAGRAM_TEXTS` の原文+現代語訳
3. AI解説: 象(trigramSymbolism)/ 古典の意味 / いまの意味 / 問いへの応用(仕事・人間関係・決断)の4小節
4. 六爻: `<details>` ×6。summary = 爻名(初九…)+原文冒頭。開くと爻辞全文+現代語訳+「この爻が動くと → 之卦◯◯」リンク(朱アクセント)
5. 関係ナビ: 互卦・錯卦・綜卦の3カード(卦画+卦名+一言説明)→ 各詳細へ。自己一致時はリンクせず注記
6. 序卦ナビ: 前の卦 / 次の卦(1と64は片側のみ)
7. 占い導線: 「問いを立てて占う」→ `/ask`、quiet link で `/formal`・`/dice`、一覧へ戻る

範囲外・非整数の番号は 404。

### 既存フローからの導線

- `HexagramResultView`(formal/dice 含む)と `InterpretationView` の本卦・之卦の卦名下に quiet link「辞典で見る」
- 公開トップの「易経・六十四卦辞典」カードから一覧へ進める

「この卦に問いかける」対話機能はスコープ外(辞典は静的)。

## 卦カード `/card`(ライト入口+SNS共有)

タグライン「いまの自分に、卦を一枚」。`IchingCardApp` のステートマシン: `pick → casting → result`。ランタイム LLM なし。

- **pick**: プリセット質問プール(`domain/cardQuestions.ts`、約120問・5カテゴリ)からランダム5枚を提示。「別の問いを見る」で再シャッフル。候補はマウント後に選ぶ(SSR不整合回避)。フッターに本編 `/ask` への quiet link
- **casting**: 選んだ瞬間に `castDice()`(変爻は常に1本)。2〜3秒の短い演出=六爻が下から積み上がり、最後に変爻が朱に灯る。formal/dice の長尺シーンは使わない。`prefers-reduced-motion` では即時表示
- **result**: 画面カード(`HexagramCard`: 問い / 卦画 / 卦名・読み / keyword / message / 変爻の爻辞現代語訳 / ブランド+タグライン)+共有パネル+CTA「この問いを、もっと深く占う → /ask」「もう一枚引く」

### 共有(`SharePanel`)

- **画像を共有**: `/api/card-image?code=…`(1080×1920 PNG)を fetch → `navigator.share({ files })`。対応環境でのみ表示(Instagram / TikTok へは共有シート経由)
- **画像を保存**: 同じ画像をダウンロード(デスクトップ向けフォールバック)
- **Xで共有**: `twitter.com/intent/tweet` に問い+卦名+共有URL
- **リンクをコピー**: 共有ランディングURL

### 共有ランディング `/card/r/[code]`

- `code = "{qid}-{hex}-{line}"`(質問ID・序卦番号1-64・変爻位置0-5)。サーバー保存なしでカードを決定的に再現。不正コードは 404
- 同じカードを表示+CTA「自分も一枚引く → /card」「本編へ → /ask」
- `generateMetadata` でタイトル(問い+卦名)と `summary_large_image`、同セグメントの `opengraph-image.tsx` が code から 1200×630 のカードOGを生成

### 既存フローからの導線

- 公開トップからの導線は設けない

## 結果画面(共通 `HexagramResultView`)

- 本卦(卦名・読み)と之卦(卦名・読み)を並べて表示。
- 変爻の位置を朱でマークし、「◯爻変」と明示。変爻なしの場合は本卦のみ(「変爻なし」)。
- 易経の引用(`HEXAGRAM_TEXTS`): 本卦の卦辞(原文+現代語訳)、変爻の爻辞(爻名「初九」等の見出し、朱のラベル)、之卦の卦辞を表示。
- 「もう一度」(新しい結果で再生)と「モード選択へ」導線。
